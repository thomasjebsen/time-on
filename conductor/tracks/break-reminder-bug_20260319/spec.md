# Specification: Break Reminder Never Fires

**Track ID:** break-reminder-bug_20260319
**Type:** Bug
**Created:** 2026-03-19
**Status:** Draft

## Summary

Break reminders never trigger. The reminder logic checks elapsed time since `lastReminderTime`, but this was constantly reset by `startNewSession()` — which was called far too often due to the idle detection bug (now fixed). Needs tests to verify the logic works correctly post-fix, and any remaining issues fixed.

## Context

Break reminders are a core feature — the app should notify users to take a break after a configurable interval (default 20 minutes). The reminder fires a system notification and shakes the menu bar icon.

## Problem Description

**Observed:** Break reminders never appear regardless of how long the user works.

**Expected:** After the configured interval (default 20 min) of continuous active use, a notification should appear and the menu bar icon should shake.

**Root Cause Hypothesis:** The previous idle detection bug caused `startNewSession()` to be called constantly, resetting `lastReminderTime = Date()` each time. The 20-minute countdown could never complete. Now that idle detection is fixed, the reminder may work — but this needs verification via tests. Additional concerns:
- Notification permission may not have been granted (silent failure)
- `onBreakReminder` callback (menu bar shake) is independent of notifications
