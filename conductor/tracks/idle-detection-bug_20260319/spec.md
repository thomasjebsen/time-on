# Specification: Idle Detection Unreliable — Timer Resets During Active Use

**Track ID:** idle-detection-bug_20260319
**Type:** Bug
**Created:** 2026-03-19
**Status:** Draft

## Summary

The session timer frequently resets to 0 even when the user is actively using the computer (typing, moving cursor). The app incorrectly detects the user as idle, ends the session, then immediately starts a new one — causing the displayed time to drop to 0.

## Context

Time On's core value proposition is accurate session time tracking displayed in the menu bar. If the timer constantly resets during active use, the app is fundamentally broken.

## Problem Description

**Observed:** Timer "refreshes to 0" during active computer use (key presses, cursor movement). Sessions rarely count up to meaningful durations.

**Expected:** Any keyboard or mouse/trackpad input should keep the session alive. The timer should only reset after the configured idle threshold (default 5 minutes) of zero input.

**Root Cause:** `IdleDetector.systemIdleTime()` uses `IOHIDSystem`'s `HIDIdleTime` property via IOKit. This API is unreliable on modern macOS (12+) — it can return stale or incorrect values, reporting the user as idle for minutes even during active input. When the reported idle time exceeds the threshold, `SessionManager.tick()` ends the current session and starts a new one on the next tick, resetting the displayed time to 0.

**Affected Code:**
- `Sources/TimeOn/IdleDetector.swift` — sole source of idle time data
- `Sources/TimeOn/SessionManager.swift:118-141` — tick logic that acts on idle time

## Acceptance Criteria

- [ ] Idle detection uses `CGEventSource.secondsSinceLastEventType()` (CoreGraphics) instead of IOKit `HIDIdleTime`
- [ ] Keyboard input (any key press) resets the idle counter
- [ ] Mouse/trackpad movement resets the idle counter
- [ ] Mouse clicks and scroll events reset the idle counter
- [ ] Timer no longer resets to 0 during active use
- [ ] Idle threshold behavior unchanged (session ends after configured minutes of true inactivity)

## Dependencies

None — self-contained fix within `IdleDetector.swift`.

## Out of Scope

- Changing the idle threshold default or session management logic
- Adding new preferences or UI
- Test infrastructure (none configured)

## Technical Notes

`CGEventSource.secondsSinceLastEventType(.hidSystemState, eventType:)` is the modern macOS API for checking idle time. It should be queried across multiple event types (key down, mouse moved, mouse clicks, scroll wheel, drag) and the minimum value used — representing time since the most recent input of any kind. This replaces the IOKit dependency for idle detection entirely.
