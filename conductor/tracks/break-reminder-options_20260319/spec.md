# Specification: Configurable Break Reminder Indicators

**Track ID:** break-reminder-options_20260319
**Type:** Feature
**Created:** 2026-03-19
**Status:** Draft

## Summary

Expand break reminders with independently configurable indicators: banner notifications, sound (with system sound selection and volume), menu bar shake (with duration and "shake until clicked"), and color-coded timer text (before/after break threshold).

## Context

Break reminders currently fire a system notification (which often fails due to ad-hoc signing) and a brief menu bar shake. Users need more control over how they're reminded, and color-coded timer text provides passive awareness without interruption.

## Acceptance Criteria

### Settings UI

- [ ] Break reminders section is its own section in Settings, below idle time setting
- [ ] Master "Enable break reminders" toggle with "Remind every: X minutes" underneath
- [ ] Each indicator is independently toggleable below the master toggle

### Banner Notification

- [ ] On/off toggle
- [ ] Enabling triggers notification permission flow (already built)
- [ ] Notification sent WITHOUT sound (`content.sound = nil`) — app sound setting takes precedence

### Sound

- [ ] On/off toggle
- [ ] Dropdown/picker listing available macOS system sounds
- [ ] Volume slider (0–100%)
- [ ] Sound plays independently of banner notification

### Menu Bar Shake

- [ ] On/off toggle
- [ ] Duration setting (seconds)
- [ ] "Shake until clicked" option — continuous shake that stops only when user clicks the menu bar item

### Color Indicators

- [ ] "Color before break" toggle (default off) with color picker (default green)
- [ ] "Color after break" toggle (default off) with color picker (default red)
- [ ] Colors apply to timer text in menu bar
- [ ] Four combos work: off/off, on/off, off/on, on/on
- [ ] Colors work as standalone indicator (no need for banner/sound/shake)
- [ ] "Before" color shows from session start until reminder interval
- [ ] "After" color shows once reminder interval has passed

## Dependencies

- Idle detection fix (merged)
- Break reminder logic (verified working)
- Notification permission flow (built)

## Out of Scope

- Custom sound files (only system sounds)
- Per-reminder snooze/dismiss
- Multiple reminder intervals

## Technical Notes

- System sounds: `NSSound` can enumerate `/System/Library/Sounds/*.aiff`
- Color in menu bar: `NSStatusBarButton.attributedTitle` with `NSAttributedString` for colored text
- Shake until clicked: continuous `CAAnimation` with `repeatCount = .greatestFiniteMagnitude`, removed on click
- All new preferences go in `Preferences.swift` as static properties backed by UserDefaults
- `SessionManager` needs to communicate "before/after break" state to `StatusBarController` for coloring
